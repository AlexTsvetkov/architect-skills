# Prompt: Lesson 00 — Cloud-Native Architect Knowledge Map

```markdown
You are an expert cloud-native software architect and technical educator. Your task is to create **Lesson 00: Cloud-Native Architect Knowledge Map** — the course overview and learning path document for a professional-level course on cloud-native architecture.

### Context about the course

This course targets experienced Java developers and team leads transitioning to cloud-native architecture roles. The audience has:
- Strong Java/Spring Boot experience
- Enterprise monolith background
- Read "System Design Interview" (Alex Xu)
- Partially read "Designing Data-Intensive Applications" (Kleppmann)

The course has 15 lessons (00-14):
```
00 - Architect Knowledge Map (this document)
01 - Cloud-Native Fundamentals
02 - Architecture Styles
03 - Microservices Deep Dive
04 - Distributed Systems & Data
05 - Containers & Orchestration
06 - Cloud-Native Patterns
07 - API Design
08 - Observability & Reliability
09 - Security
10 - Event-Driven Architecture
11 - CI/CD & GitOps
12 - Scalability & Performance
13 - Architecture Decisions
14 - Cost Optimization
```

### Requirements

Create a comprehensive knowledge map that serves as the course's entry point and navigation hub. Include:

1. **Course Introduction** — Who this is for, what they'll learn, how to use the guide
2. **Knowledge Baseline Assessment** — Table mapping the reader's existing knowledge to course topics
3. **Course Structure** — 6 parts grouping lessons thematically:
   - Part I: Foundations (Lessons 01-03)
   - Part II: Distributed Systems & Data (Lessons 04, 10)
   - Part III: Cloud Infrastructure (Lessons 05, 06, 08)
   - Part IV: Security & APIs (Lessons 07, 09)
   - Part V: Delivery & Operations (Lessons 11, 12)
   - Part VI: Architecture Practice (Lessons 13, 14)
4. **Lesson Summaries** — For each lesson:
   - File path reference
   - Bullet list of key topics covered (8-12 items)
   - "Connection to your experience" note bridging enterprise Java to cloud-native
   - Key exercises mentioned
5. **Recommended Reading Path** — Books organized by priority:
   - Must-Read (5 books with why/when)
   - Highly Recommended (5 books)
   - Deep Dives (5 books)
6. **Online Resources** — Table with 8-10 authoritative URLs
7. **Certifications** — Table with 5 relevant certifications
8. **Learning Schedule** — Week-by-week plan (28 weeks) mapping lessons to parallel reading

### Style
- Use tables for structured data
- Use ASCII diagrams where helpful
- Keep descriptions concise but informative
- Include file path references for each lesson
- Add creation/update timestamps
